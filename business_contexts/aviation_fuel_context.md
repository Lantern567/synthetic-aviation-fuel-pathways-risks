# 航空燃料业务上下文

## 业务概述

航空燃料业务专注于传统航空燃料和绿色甲醇在航空运输中的应用分析，包括燃料消耗计算、碳排放评估、成本效益分析等。

## 核心技术栈

### pyBADA燃料计算系统
- **版本**：pyBADA v0.1.5 + EUROCONTROL BADA3/4
- **计算模型**：完整三阶段飞行轨迹物理建模（爬升-巡航-下降）
- **精度要求**：误差 < 5%，支持136个机型的精确建模
- **性能指标**：单航线计算 < 1秒，包含缓存优化

### 机型支持范围
- **窄体机**：A320/A319/A321, B737-700/800/900, C919, ARJ21
- **宽体机**：A330/A340/A350, B777/B787, B747
- **支线机**：CRJ900, E190/E195, ATR72
- **货机**：B747F, A330F, B777F

## 计算标准和方法

### 距离计算
- **公式**：Haversine大圆距离公式
- **精度**：±1km (航线距离 > 500km)
- **修正因子**：高原机场 +5-10%，逆风 +3-8%

### 燃料消耗计算层级
1. **完整轨迹模型** (优先)：基于BADA3物理建模
2. **简化轨迹模型** (备用)：线性插值近似
3. **经验公式** (兜底)：基于距离和机型的经验公式

### 碳排放计算标准
- **直接排放**：燃料燃烧产生的CO2
- **间接排放**：燃料生产和运输的碳足迹
- **生命周期评估**：Well-to-Wake完整碳足迹
- **国际标准**：符合ICAO和EU ETS要求

## 业务流程和工作流

### 标准燃料分析流程
1. **数据准备**：加载和验证航班数据
2. **机型匹配**：ICAO代码映射到技术参数
3. **燃料计算**：pyBADA完整轨迹计算
4. **碳排放评估**：基于燃料类型和排放因子
5. **成本分析**：燃料成本和运营成本
6. **对比分析**：传统燃料 vs 绿色甲醇
7. **结果输出**：报表、图表、汇总统计

### 质量保证检查点
```yaml
计算前检查:
  - pyBADA环境可用性
  - 机型参数完整性
  - 输入数据格式正确性

计算中监控:
  - 计算成功率 (目标: >95%)
  - 性能指标 (单航班 <1秒)
  - 内存使用 (避免溢出)

计算后验证:
  - 结果合理性范围检查
  - 已知基准对比验证
  - 异常值识别和处理
```

## 常见问题和解决方案

### 技术问题
1. **XML解析错误**
   - 原因：BADA机型数据文件损坏
   - 解决：使用DUMMY通用模型，跳过XML解析
   - 影响：精度略降但保证100%计算成功率

2. **机型不支持**
   - 原因：新机型或不常见机型
   - 解决：映射到相似机型或使用通用模型
   - 备选：基于座位数和航程的估算

3. **性能优化**
   - 并行计算：使用multiprocessing处理大数据集
   - 缓存机制：重复计算结果缓存
   - 分块处理：大文件分批次处理

### 业务问题
1. **绿色甲醇数据缺乏**
   - 解决：基于理论计算和实验数据估算
   - 参考：国际可再生能源署(IRENA)数据
   - 更新：跟踪最新研究和工业数据

2. **成本变动频繁**
   - 解决：建立价格跟踪机制
   - 工具：价格指数和趋势分析
   - 策略：情景分析和敏感性测试
