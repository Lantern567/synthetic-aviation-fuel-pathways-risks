## 1. 建模概述

### 1.1 问题描述

针对多个机场的绿色甲醇/航煤供应需求，建立多种供应链模式的**并行独立优化模型**。每种供应链模式单独建模和求解，获得各自的最优配置方案，而非在供应链间进行选择。最终输出每个供应链的最优成本、配置和运营策略，为决策者提供全面的供应链方案对比。

### 1.2 供应链模式

本研究涉及三种并行独立的供应链模式：

1. **天然气基供应链** - 基于天然气和氢气的混合原料路径
2. **生物质供应链** - 基于生物质原料的分布式生产模式  
3. **绿电基供应链** - 基于可再生能源的电制甲醇路径

### 1.3 建模策略

- **独立建模原则**：每个供应链作为独立的优化问题求解
- **统一目标函数**：各供应链均以最小化经济成本为目标
- **标准化输出**：确保结果可比较和分析
- **并行计算**：利用多核/集群进行并行求解

### 1.4 评估指标

采用3E指标体系进行综合评估：
- **经济性（Economy）**：总成本最小化
- **环境性（Environment）**：碳排放评估  
- **能源安全（Energy Security）**：供应可靠性评估

## 2. 数学建模框架

### 2.1 总体建模架构

#### 2.1.1 并行独立优化模型

每个供应链 s ∈ S = {天然气基, 生物质, 绿电基} 构建独立的优化模型：

```
Model_s: Min Z_s = f_s(X_s, Y_s, W_s)
Subject to: g_s(X_s, Y_s, W_s) ≤ 0
           h_s(X_s, Y_s, W_s) = 0
           X_s, Y_s ≥ 0, W_s ∈ {0,1}
```

其中：
- s: 供应链类型标识
- X_s: 供应链s的生产决策变量
- Y_s: 供应链s的运输决策变量  
- W_s: 供应链s的设施建设决策变量
- Z_s: 供应链s的总成本

#### 2.1.2 统一约束框架

虽然供应链独立优化，但共享统一的约束框架：

**机场需求约束（跨供应链一致）**：
```
∑∑ Y_s,ijkm = D_kt, ∀k,t,s  (每个供应链都需满足相同的机场需求)
```

**供应链内部约束（独立）**：
```
供应链s内: ∑∑∑ Y_s,ijkm ≤ ∑ X_s,ijt, ∀i,t  (生产-运输平衡)
```

### 2.2 优化目标函数

#### 2.2.1 各供应链独立目标函数

**天然气基供应链 (s=NG)**：
```
Min Z_NG = ∑∑∑ (NG_cost_ijt × NG_ijt) + ∑∑∑ (H2_cost_ijt × H2_ijt) + 
           ∑∑∑ (PC_NG_ijt × X_NG_ijt) + ∑∑∑∑ (TC_NG_ijkm × Y_NG_ijkm) + 
           ∑∑ (FC_NG_ij × W_NG_ij)
```

**生物质供应链 (s=BM)**：
```
Min Z_BM = ∑∑∑ (PC_BM,ijt × X_BM,ijt) + ∑∑∑∑ (TC_BM,ijkm × Y_BM,ijkm) + 
           ∑∑ (FC_BM,ij × W_BM,ij) + ∑∑∑ (Collection_cost × BR_BM)
```

**绿电基供应链 (s=GE)**：
```
Min Z_GE = ∑∑∑ (PC_GE,ijt × X_GE,ijt) + ∑∑∑∑ (TC_GE,ijkm × Y_GE,ijkm) + 
           ∑∑ (FC_GE,ij × W_GE,ij) + ∑∑ (electricity_cost × GE_consumption)
```

#### 2.2.2 最终输出

优化求解后获得三个独立最优解：
```
最优解集合 = {
    (Z_NG*, X_NG*, Y_NG*, W_NG*),  # 天然气基最优方案
    (Z_BM*, X_BM*, Y_BM*, W_BM*),  # 生物质基最优方案  
    (Z_GE*, X_GE*, Y_GE*, W_GE*)   # 绿电基最优方案
}
```

### 2.3 决策变量定义

#### 2.3.1 通用变量结构

所有供应链共享的变量结构：
- **生产变量 X_{s,ijt}**：供应链s在地点i使用技术j在时期t的生产量
- **运输变量 Y_{s,ijkm}**：供应链s从地点i到机场k使用运输方式m的运输量
- **设施变量 W_{s,ij}**：供应链s在地点i是否建设技术j设施

#### 2.3.2 供应链特定变量

各供应链根据自身特点定义专用变量（详见第3章分供应链建模）。

### 2.4 约束条件框架

#### 2.4.1 通用约束

**需求满足约束**：
```
∑∑∑ Y_s,ijkm ≥ D_kt, ∀k,t,s
```

**生产平衡约束**：
```
∑∑ Y_s,ijkm ≤ X_s,ijt, ∀i,j,t,s
```

**设施建设约束**：
```
X_s,ijt ≤ M × W_s,ij, ∀i,j,t,s
```

#### 2.4.2 供应链特定约束

各供应链根据技术特点和资源特性定义专用约束条件。

### 3.1 天然气基供应链模型

#### 3.1.1 原材料决策空间

根据不同的供应链模式，设置不同的情景，每种情景下的决策变量为：

**天然气源头：**
- **管段供应**：从天然气管段里直接获取，每个管段每天的价格动态变化
- **LNG接收站**：价格相对稳定统一

**氢气源头：**
- **光伏制氢**：基于中国光伏电站的真实发电数据（3小时级别数据）
- **直接采购**：市场采购

**氢能产地决策空间**：所有的风电和光伏发电厂  
**制备航煤地点的决策空间**：氢能产地、机场

#### 3.1.2 运输模式设定

**5种运输模式：**
1. **模式①**：天然气管段→氢气产地→直接制备航煤（氢田直供）

3. **模式③**：LNG接收站→制备航煤（港口进口）
4. **模式④**：LNG接收站→氢气产地→制备航煤（港口转运）
5. **模式⑤**：两者一起→机场→制备航煤（综合供应）

#### 3.1.5 时间尺度匹配建模 ⭐

#### 3.1.5.1 多时间尺度问题

天然气基供应链中存在显著的时间尺度差异：

**生产侧时间尺度**：
- 太阳能发电：1小时粒度数据
- 氢气制取：实时响应发电功率变化
- 航煤生产：小时级调节能力

**需求侧时间尺度**：
- 机场燃料定制：以周为单位制定计划
- 航班调度：周期性需求模式
- 运输安排：周级批次运输

#### 3.1.5.2 时间尺度协调机制

```python

# 时间映射函数
def hour_to_week(hour_index):
    return hour_index // 168

def week_to_hour_range(week_index):
    return range(week_index * 168, (week_index + 1) * 168)
```

#### 4.2.2 大规模问题分解算法

```python
class BendersDecomposition:
    def __init__(self, master_vars, subproblem_vars):
        self.master_vars = master_vars
        self.subproblem_vars = subproblem_vars
        self.tolerance = 1e-6
        self.max_iterations = 100
    
    def solve(self):
        # 初始化主问题
        master_problem = self.create_master_problem()
        
        iteration = 0
        while iteration < self.max_iterations:
            # 求解主问题
            master_solution = master_problem.optimize()
            
            # 求解子问题
            subproblem_solutions = self.solve_subproblems(master_solution)
            
            # 检查收敛性
            if self.check_convergence(master_solution, subproblem_solutions):
                break
            
            # 添加Benders割
            self.add_benders_cuts(master_problem, subproblem_solutions)
            
            iteration += 1
        
        return self.construct_full_solution(master_solution, subproblem_solutions)
```

```python
        # 第二阶段：运营决策（针对每个情景）

        for scenario in self.scenarios:
            second_stage_solution = self.solve_second_stage(scenario, first_stage_vars)
            second_stage_solutions.append(second_stage_solution)
        
        # 计算期望成本
        expected_cost = self.calculate_expected_cost(second_stage_solutions)
        
        return {
            'first_stage': first_stage_vars,
            'second_stage': second_stage_solutions,
            'expected_cost': expected_cost
        }
```

```python
def production_cost(capacity, technology):
    # 规模经济效应
    scale_factor = (capacity / reference_capacity) ** economies_of_scale_exponent
    
    # 年化资本成本
    annualized_capex = technology['capex'] * capital_recovery_factor
    
    # 运营成本
    opex = technology['opex'] * capacity
    
    return (annualized_capex + opex) * scale_factor
```
