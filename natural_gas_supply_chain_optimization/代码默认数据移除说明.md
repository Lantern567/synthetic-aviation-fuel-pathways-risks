# 代码默认数据移除说明

## 概述

本次修改彻底移除了代码中的所有默认数据和示例数据，使代码完全依赖真实的数据文件。这一改动提高了代码的可靠性和数据准确性，但也意味着所有必要的数据文件都必须存在且格式正确。

## 主要修改内容

### 1. 移除的默认数据类型

#### 1.1 可再生能源默认数据
- **移除**：`_create_fallback_renewable_data()` 方法
- **影响**：当可再生能源数据目录不存在或数据文件缺失时，程序将抛出 `FileNotFoundError` 而不是使用模拟数据
- **要求**：必须提供真实的风电和太阳能发电数据文件

#### 1.2 机场默认数据
- **移除**：CSV数据解析失败时的默认机场列表
- **移除**：机场燃油需求数据缺失时的默认值 (1000kg/周)
- **影响**：所有机场必须有完整的 `weekly_fuel_series` 数据
- **要求**：机场数据必须包含完整的52周燃油需求时间序列

#### 1.3 天然气管道默认数据
- **移除**：`_generate_example_pipeline_data()` 方法
- **移除**：管道容量和长度的默认值
- **移除**：天然气价格的默认值
- **影响**：必须有真实的管道GIS数据文件
- **要求**：管道数据必须包含容量、长度、价格等完整信息

#### 1.4 LNG接收站默认数据
- **移除**：`_generate_example_lng_data()` 方法
- **移除**：接收站容量和存储容量的默认值
- **影响**：必须有真实的LNG接收站数据文件
- **要求**：LNG数据必须包含容量、存储等完整信息

### 2. 错误处理改进

#### 2.1 数据文件缺失处理
```python
# 修改前：使用默认数据
if not os.path.exists(data_file):
    logger.warning("数据文件不存在，使用默认数据")
    return self._create_default_data()

# 修改后：抛出异常
if not os.path.exists(data_file):
    logger.error(f"数据文件不存在: {data_file}")
    raise FileNotFoundError(f"无法找到数据文件: {data_file}")
```

#### 2.2 数据质量检查
```python
# 修改前：使用默认值
value = data.get('field', default_value)

# 修改后：严格检查
value = data.get('field', None)
if value is None:
    logger.error(f"缺少必要字段: field")
    raise ValueError(f"数据不完整，缺少字段: field")
```

## 对用户的影响

### 3.1 正面影响
1. **数据准确性提升**：不再使用不准确的模拟数据
2. **结果可靠性**：优化结果基于真实数据，更具可信度
3. **问题早期发现**：数据问题在运行初期就会被发现，而不是产生错误的结果

### 3.2 需要注意的事项
1. **数据文件要求**：所有数据文件必须存在且格式正确
2. **错误提示**：数据问题会导致程序终止，但会提供明确的错误信息
3. **调试难度**：需要确保所有数据源都可用
